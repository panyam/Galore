Parse Tables

Parse tables are the core data structure used by LR parsers. They encode the state machine that drives the parsing process.

Table Structure

An LR parse table consists of two parts:

ACTION Table

Maps (state, terminal) pairs to actions:

  • Shift n: Push the terminal and go to state n
  • Reduce r: Apply grammar rule r
  • Accept: Parsing complete
  • Error: Syntax error

GOTO Table

Maps (state, non-terminal) pairs to states. Used after reductions.

Viewing Parse Tables

Galore provides utilities to visualize parse tables:

import * as G from "galore";

const parser = G.newParser(grammar, { type: "lalr" });

// Get HTML representation
const html = G.Printers.parseTableToHtml(parser.parseTable);

// Get text representation
const text = G.Printers.parseTableToText(parser.parseTable);

Understanding Conflicts

When a parse table cell has multiple actions, you have a conflict:

  • Shift-Reduce Conflict: Grammar is ambiguous about whether to shift or reduce
  • Reduce-Reduce Conflict: Grammar is ambiguous about which rule to apply

Use the Playground to visualize parse tables and identify conflicts.
